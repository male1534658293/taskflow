# TaskFlow 发布流程指南

## 一、首次发布（只需做一次）

### 1. 生成 SSH Key（用于免密推送代码）

```bash
ssh-keygen -t ed25519 -C "your@email.com" -f ~/.ssh/id_ed25519 -N ""
cat ~/.ssh/id_ed25519.pub  # 复制输出的内容
```

去 GitHub 添加公钥：
**GitHub → Settings → SSH and GPG keys → New SSH key**
粘贴上面复制的内容，保存。

验证是否成功：
```bash
ssh-keyscan github.com >> ~/.ssh/known_hosts
ssh -T git@github.com
# 看到 "Hi xxx! You've successfully authenticated" 即成功
```

---

### 2. 在 GitHub 创建仓库

1. 打开 [https://github.com/new](https://github.com/new)
2. Repository name 填 `taskflow`
3. **不要**勾选 "Add a README file"
4. 点 **Create repository**

---

### 3. 添加 GitHub Secrets（Google 日历同步用）

打开：**仓库页面 → Settings → Secrets and variables → Actions → New repository secret**

添加两条：

| Name | Secret |
|------|--------|
| `VITE_GOOGLE_CLIENT_ID` | 你的 Google Cloud Client ID |
| `VITE_GOOGLE_CLIENT_SECRET` | 你的 Google Cloud Client Secret |

---

### 4. 初始化本地仓库并推送代码

```bash
cd /Users/bytedance/Desktop/Components

# 确保有 .gitignore（排除 node_modules 等大文件）
# 项目中已有，无需重复操作

git init
git config user.email "your@email.com"
git config user.name "your-github-username"
git add .
git commit -m "TaskFlow v1.0.0 initial release"
git remote add origin git@github.com:male1534658293/taskflow.git
git branch -M main
git push -u origin main
```

---

### 5. 打 Tag 触发自动构建

```bash
git tag v1.0.0
git push origin v1.0.0
```

GitHub Actions 自动构建 `.dmg`，约 5-10 分钟后出现在：
👉 https://github.com/male1534658293/taskflow/releases

---

## 二、后续更新流程（每次发新版本）

### 第一步：修改代码

正常修改代码、修复 bug、添加功能。

---

### 第二步：更新版本号

编辑 `package.json`，把版本号改成新版本：

```json
"version": "1.1.0"
```

同时更新 `CHANGELOG.md`，记录本次更新内容：

```markdown
## v1.1.0 (2026-04-01)

### 新功能
- 添加了 xxx 功能

### 修复
- 修复了 xxx bug
```

---

### 第三步：提交代码

```bash
git add .
git commit -m "Release v1.1.0"
git push origin main
```

---

### 第四步：打 Tag 触发自动构建

```bash
git tag v1.1.0
git push origin v1.1.0
```

GitHub Actions 自动构建并发布新版本到 Releases。

已安装旧版本的用户，下次打开 TaskFlow 时会自动收到更新提示，点击重启即可完成升级。

---

## 三、常用命令速查

| 操作 | 命令 |
|------|------|
| 查看改动了哪些文件 | `git status` |
| 查看具体改动内容 | `git diff` |
| 提交所有改动 | `git add . && git commit -m "描述"` |
| 推送到 GitHub | `git push origin main` |
| 查看所有 tag | `git tag` |
| 发布新版本 | `git tag v1.x.x && git push origin v1.x.x` |
| 删除错误的 tag | `git tag -d v1.x.x && git push origin --delete v1.x.x` |

---

## 四、查看构建状态

- **构建进度**：https://github.com/male1534658293/taskflow/actions

## 五、分发给他人

构建成功后，把 Releases 页面链接发给对方：

```
https://github.com/male1534658293/taskflow/releases/latest
```

对方下载 `.dmg` 文件，双击安装即可。

> **首次打开提示"未经验证的开发者"**：右键点击应用 → 打开 → 打开
